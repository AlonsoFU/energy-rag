1. HÍBRIDO ES EL ESTÁNDAR (95% producción 2025)
   ✅ BM25 + Dense retrieval obligatorio

2. RRF ES LA FUSIÓN GANADORA
   ✅ Reciprocal Rank Fusion (k=60)
   ✅ NO requiere tuning (vs weighted fusion)
   ✅ Score-agnostic (maneja escalas diferentes)
   ⚠️  Solo 3.86% peor que weighted con tuning

3. 3-WAY RETRIEVAL ES ÓPTIMO (IBM Research)
   ✅ BM25 + Dense + Sparse (SPLADE)
   ✅ +13.8% mejora vs solo dense
   ✅ Mejores resultados en benchmarks 2025

4. RERANKING ES OBLIGATORIO
   ✅ ColBERT v2: Balance perfecto (180× más eficiente)
   ✅ Cross-encoder: Máxima precisión (+3-5%, pero 10× más lento)
   ✅ +15-30% recall adicional

5. SEMANTIC CHUNKING GANA
   ✅ Page-level: 64.8% accuracy (NVIDIA benchmark #1)
   ✅ Semantic: +9% recall vs fixed chunks
   ✅ 400-512 tokens: Sweet spot (85-90% recall Chroma)

AJUSTES RECOMENDADOS:
════════════════════════════════════════════════════════════════════════════

CAMBIO 1: Usar RRF en lugar de Weighted Fusion ✅ PRIORITARIO

  Antes:
    score_final = α * score_keywords + (1-α) * score_embeddings
    (Requiere tuning de α)

  Ahora:
    def rrf(rank, k=60):
        return 1 / (k + rank)

    score_final = rrf(rank_bm25) + rrf(rank_dense)
    (k=60 funciona sin tuning)

  Razón:
    • Más simple (no requiere tuning)
    • Estándar industria 2025
    • Solo 3.86% peor que weighted


CAMBIO 2: Agregar ColBERT Reranking (Opcional) ⚠️

  Pipeline completo:
    1. BM25 + Dense → Top 100
    2. RRF fusion → Top 50
    3. ColBERT reranker → Top 10

  Ganancia: +15-30% recall
  Costo: +60-150ms latencia
  Cuándo: Si precisión >85% es crítica


CAMBIO 3: Evaluar e5-large si crece ⚠️

  Actual: jina-es (62.1 MTEB)
  Considerar: e5-large-instruct (67.3 MTEB)

  Trade-off:
    jina-es:     Especializado español, 8K context, 137M params
    e5-large:    +8% mejor, 100 idiomas, 560M params (más pesado)

  Decisión: Mantener jina-es HASTA >10K normas


CAMBIO 4: 3-Way Retrieval si dataset crece ⚠️

  Cuando: >20K normas
  Agregar: SPLADE sparse vectors
  Ganancia: +5-8% recall adicional
  Costo: +15ms latencia

FASE 2 (2026-2027, >5K normas): Híbrido RRF
──────────────────────────────────────────────────────────────────────────

  Stack:
    1. BM25 sobre títulos → Top 100 (15ms)
    2. Dense (jina-es) sobre títulos enriquecidos → Top 100 (45ms)
    3. RRF fusion (k=60) → Top 50 (5ms)              ← NUEVO
    4. Metadata expert re-ranking → Top 10 (5ms)

  Código:
    ```python
    # RRF fusion
    def rrf_fusion(rankings, k=60):
        scores = defaultdict(float)
        for ranking in rankings:
            for rank, doc_id in enumerate(ranking, 1):
                scores[doc_id] += 1 / (k + rank)
        return sorted(scores.items(), key=lambda x: x[1], reverse=True)

    # Uso
    bm25_results = bm25.search(query, top_k=100)
    dense_results = embeddings.search(query, top_k=100)
    fused = rrf_fusion([bm25_results, dense_results])
    ```

  Precisión: 86%
  Latencia: 70ms
  Costo: $0
  Esfuerzo: 1 semana

  ✅ IMPLEMENTAR cuando dataset >5K
