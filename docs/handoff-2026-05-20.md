## Causa raíz del cuelgue de Ollama (resuelta)

Ollama `qwen3.5:9b` deadlockea cuando recibe JSON-schema-constrained
decoding combinado con prompts grandes y/o el flag `think=false` (que SÍ
usábamos). Confirmado por:

- Probe sistemático: TODOS los JSON schemas cuelgan, sin importar contenido
  ni tamaño del enum (incluso 2 entradas con dígitos limpios).
- Sin schema, generación plana funciona perfecto.
- Issues upstream que documentan exactamente nuestro caso:
  [#15540](https://github.com/ollama/ollama/issues/15540) (qwen3.5 no enforced),
  [#15260](https://github.com/ollama/ollama/issues/15260) (think=false rompe format),
  [#15502](https://github.com/ollama/ollama/issues/15502) (repetition loop),
  [#8063](https://github.com/ollama/ollama/issues/8063) (general).

**Fix aplicado**: **patrón híbrido** — generación libre + `verify_citations`
post-hoc + retry con prompt estricto. Industria-recomendado cuando el
constrained-decoding del runtime no es confiable.

## ADDENDUM 2026-05-22 — Fase 0 cerrada: cifra dura de correctitud

Branch `feat/fase0-correctness` (off main, post-merge PR #4). Implementado el
chequeo de correctitud que el roadmap pedía como criterio de aceptación de

### Qué mide (`src/eval/correctness.py`, funciones puras + 10 tests)
Por cada query in_domain:
- **`cited_expected`** (binario, EXACTO): ¿la respuesta cita el artículo

`scripts/score_correctness.py` consume un results JSON y mapea cada query a su
`conceptos.definicion` con la MISMA normalización que el inject. Off_corpus se
puntúa aparte (refusal). Correr: `PYTHONPATH=. ./venv/bin/python
scripts/score_correctness.py --results data/eval/results/<ts>.json`.

### ADDENDUM 2026-05-23b — bug de acrónimos: causa raíz + fix (cerrado)
Debugging sistemático del rechazo de acrónimos (SEC/CEN/SEN…).

**alias/sigla**, el inject resuelve y pone el artículo al tope, pero ese artículo

`inject_definition` — cuando el término matcheado ≠ nombre canónico, inyecta un
chunk que dice el vínculo explícito «alias» se refiere a «canónico». {def}
(verbatim, datos curados). Scoped a queries-por-alias → **sin riesgo de las
regresiones de colisión** (esas eran queries por nombre canónico). `_concept_index`
y `find_curated_definition` ahora cargan el nombre canónico (4-tupla).
**Resultado** (categoría `alias_sigla`, 41q): answered 78→**97%**, cita_ok
63→**95%**, 8 recuperadas, **0 regresiones**. 8 tests unitarios nuevos/actualizados.

### ADDENDUM 2026-05-24b — Inyección disparada por concepto (no por fraseo) (EJECUTADO)

Plan `2026-05-24-concept-triggered-injection.md`, rama `feat/definition-source-resolver`.

```
UA='Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 ... Chrome/120 Safari/537.36'
curl -A "$UA" -H 'Referer: https://www.bcn.cl/leychile' \
  'https://www.bcn.cl/leychile/Consulta/obtxml?opt=7&idNorma=258171' -o data/raw/lgse_258171_obtxml.xml
```
Devuelve **893KB de XML estructurado** (`EsquemaIntercambioNorma`): cada artículo es un
`<EstructuraFuncional>` cuyo `<Texto>` empieza con "Artículo N°.-". Trae `<FechaDerogacion>`
(bonus para vigencia/B2). XML guardado en `data/raw/lgse_258171_obtxml.xml`.
